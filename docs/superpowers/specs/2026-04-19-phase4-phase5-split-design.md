# Phase 4 / Phase 5 拆分设计

## 背景

Phase 3 已全部完成。游戏目前是单关卡、单轮次原型，具备完整的核心玩法（三消、断链/并链、连锁、combo 计分）、精细的视觉表现（石蛙、石球、粒子特效、玛雅主题 HUD）、程序化音效和完整的 HUD。

原 ZUMA_PLAN.md 中 Phase 4 定义为"内容扩展"，包含 4 项：多关卡、特殊球、难度曲线、本地存档。本设计将这 4 项拆分成 Phase 4（最小可玩游戏闭环）和 Phase 5（玩法扩展）。

## 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| MVP 标准 | 8 关完整通关流程 | 玩家能从第 1 关玩到第 8 关，有进度感和结局 |
| 路径多样性 | 多种路径形状 | 不同关卡使用不同路径（螺旋、S 形、回字形、之字形等） |
| 特殊球归属 | 全部放 Phase 5 | Phase 4 只用普通球，减少变量，专注闭环验证 |
| 实施策略 | 方案 B（闭环优先） | 先用参数化螺旋线搭完整 8 关流程，再替换多样路径 |

## Phase 4：多关卡流程闭环

### 目标

将当前单关卡原型升级为 8 关完整可玩游戏，有关卡选择、难度递进、进度存档和多种路径形状。

### 不纳入 Phase 4 的内容

- 特殊球（任何类型）
- 暂停系统
- 背景音乐
- PWA 支持
- 9+ 关卡

### 执行顺序

```
M (数据模型) → N (选关界面+切换) → O (难度曲线) → P (存档)
                                                      ↓
                                          Q (路径泛化) → R (路径分配+打磨)
```

M→N→O→P 是"先用参数化螺旋线搭出完整 8 关流程"的主线。完成后即有可玩的 8 关游戏（路径全是螺旋线变体）。Q→R 是"多路径形状"增强线，在流程跑通后执行。

### 工作包 M：关卡数据模型 + 配置系统

**目标**：定义关卡数据结构，把当前 config.js 中的硬编码参数抽成按关卡独立配置。

**任务**：
- 新建 `levels.js` 模块，导出关卡配置数组
- 每关配置包含：
  - `id`: 关卡编号（1-8）
  - `name`: 关卡名称（玛雅风格）
  - `chainCount`: 初始球数
  - `chainSpeed`: 球链推进速度
  - `colorCount`: 使用颜色数（3-5）
  - `pathType`: 路径类型标识符
  - `pathParams`: 路径生成参数对象
  - `shooterPos`: 射手位置 `{x, y}`（可选，默认屏幕中央偏下）
  - `goalPos`: 目标/吞口位置 `{x, y}`（可选，由路径终点推导）
- `ZumaGame` 新增 `currentLevel` 字段
- `resetRound()` 从 `currentLevel` 配置读取参数，不再使用全局常量
- 向后兼容：不传关卡配置时退回当前默认值

**验收标准**：
- 修改关卡配置后 `resetRound()` 加载对应参数，游戏体验随之改变
- 切换 `currentLevel` 后状态正确重置

### 工作包 N：关卡选择界面 + 切换逻辑

**目标**：增加关卡选择画面，实现关卡间的完整切换流程。

**任务**：
- 新增 `gameState: "levelSelect"` 状态
- Canvas 上绘制关卡选择界面：
  - 8 个关卡按钮，延续石质/玛雅主题
  - 每关显示编号、名称、锁定/已解锁/已通关状态、最高分
- 交互流程：
  - 点击已解锁关卡 → 加载配置 → 进入 `playing`
  - 通关 → 结算画面 → 解锁下一关 → "继续下一关" / "返回选关"
  - 失败 → 结算画面 → "重试" / "返回选关"
  - 最后一关通关 → 全通关画面
- 游戏启动时进入 `levelSelect`（不再直接开始游戏）
- 新增 HUD "返回选关" 按钮（游戏过程中可用）

**验收标准**：
- 首次打开看到选关界面，第 1 关解锁
- 通关后下一关自动解锁
- 任何时候可返回选关
- 切换关卡不残留上一关的球链、粒子、音效、splitState、pendingMatchChecks

### 工作包 O：8 关难度曲线设计

**目标**：配置 8 关的具体参数，形成从简单到有挑战的递进体验。

**任务**：
- 设计 8 关参数递进表（暂定值，待实际测试调整）：

| 关卡 | 名称 | 球数 | 球速 | 颜色数 | 路径难度 |
|------|------|------|------|--------|----------|
| 1 | 石阶祭坛 | 20 | 52 | 3 | 简单螺旋 |
| 2 | 密林通道 | 24 | 58 | 3 | 简单变体 |
| 3 | 月牙河谷 | 28 | 64 | 4 | 中等 |
| 4 | 祭司回廊 | 30 | 68 | 4 | 中等 |
| 5 | 羽蛇阶梯 | 32 | 72 | 4 | 中等偏难 |
| 6 | 太阳神殿 | 35 | 76 | 5 | 中难 |
| 7 | 星象迷宫 | 38 | 80 | 5 | 难 |
| 8 | 黄金祭坛 | 42 | 86 | 5 | 最难 |

- 前两关作为教学关（少球、慢速、3 色）
- 中间关递增引入 4 色
- 后期关 5 色 + 高速 + 多球
- 每关名称为玛雅风格，增加氛围

**验收标准**：
- 第 1 关新手 1-2 次内通关
- 第 8 关对熟练玩家有挑战
- 相邻关卡无断崖式跳跃

### 工作包 P：关卡进度 + 本地存档

**目标**：通关进度持久化到 localStorage。

**任务**：
- 存档数据结构：
  ```json
  {
    "version": 1,
    "unlockedLevel": 3,
    "levels": {
      "1": { "cleared": true, "highScore": 12400 },
      "2": { "cleared": true, "highScore": 8900 },
      "3": { "cleared": false, "highScore": 0 }
    }
  }
  ```
- 通关时自动保存（更新 `cleared`、`highScore`、`unlockedLevel`）
- 启动时读取存档恢复状态
- 存档版本号字段，未来扩展时可做迁移
- "重置进度" 入口（关卡选择界面中提供）
- 存档损坏时安全降级为初始状态

**验收标准**：
- 通关第 3 关后刷新页面，前 3 关已通关、第 4 关已解锁
- 损坏/缺失存档时安全降级
- 重置后回到初始状态

### 工作包 Q：路径系统泛化

**目标**：将 path.js 中硬编码的阿基米德螺旋线改为可插拔的路径生成系统。

**任务**：
- 定义路径生成器接口：
  ```javascript
  // 输入：路径参数对象
  // 输出：pathPoints[]，每个元素包含 { x, y, len }
  function generatePath(pathType, pathParams) → pathPoints[]
  ```
- 实现至少 4 种路径生成器：
  1. **螺旋型**（spiral）— 当前已有，参数化圈数/半径/中心点
  2. **S 形蛇行**（serpentine）— 多段贝塞尔曲线上下蜿蜒
  3. **回字形方螺旋**（rectangular）— 圆角矩形逐层收缩
  4. **之字形锯齿**（zigzag）— 横向来回折叠下行
- 所有生成器输出兼容的 `pathPoints[]`（含 `totalPathLength`），下游 chain/projectile/render 无需改动
- 关卡配置通过 `pathType` + `pathParams` 选择路径
- 射手位置和目标点可随路径配置变化
- path.js 中的 `createPath()` 改为调度器，根据 `pathType` 委托到对应生成器

**验收标准**：
- 4 种路径在游戏中正常运行（行进、碰撞、消除均正确）
- 切换路径不影响规则正确性
- 每种路径视觉上有明显区别

### 工作包 R：8 关路径分配 + 通关流程打磨

**目标**：为 8 关分配独特路径，打磨完整通关体验。

**任务**：
- 8 关路径分配（暂定，待测试调整）：
  - 1-2 关：简单螺旋变体（熟悉操作）
  - 3-4 关：S 形蛇行（引入弯道策略）
  - 5-6 关：回字形或之字形（空间利用变化）
  - 7-8 关：混合或高复杂度路径
- 微调每关路径参数（弯度、长度、起终点位置）
- 关卡选择界面增加路径缩略图预览（迷你路径轮廓图）
- 全通关画面：祝贺 + 总分汇总 + 每关成绩回顾
- 关卡间过渡动画（简单淡入淡出）

**验收标准**：
- 8 关路径各不相同
- 视觉和难度递进合理
- 全通关有明确结尾反馈
- 从第 1 关到第 8 关有完整"通关旅程感"

## Phase 5：玩法扩展

### 目标

在 Phase 4 的多关卡框架上叠加特殊球机制和更多内容。Phase 5 的详细规格待 Phase 4 完成后再正式制定。

### 工作包概览

#### 工作包 S：特殊球系统框架

- 通用注册/触发/特效架构
- 球数据模型扩展 `specialType` 字段
- 触发时机定义（命中触发 / 消除触发 / 被动效果）
- 效果接口 `onActivate(game, ball, context)`
- 特殊球视觉标识（叠加图标 / 发光 / 特殊纹理）
- 关卡配置增加特殊球出现概率和类型白名单

#### 工作包 T：核心特殊球实现

按优先级排列：
1. **爆炸球** — 消除时摧毁半径内所有球
2. **变色球 / 彩虹球** — 匹配任意颜色
3. **倒退球** — 命中后球链暂时后退
4. **减速球** — 命中后球链速度减半持续数秒

#### 工作包 U：扩展关卡（9-12+）

- 利用特殊球设计策略关卡
- 更多路径形状变体
- 更高难度挑战关

#### 工作包 V：体验完善

- 暂停系统（完整暂停/继续）
- 背景音乐（程序化或外部 BGM）
- PWA 支持（Service Worker 离线缓存、manifest 可安装）
- 更多路径类型

## 技术注意事项

### 对现有架构的影响

1. **config.js** — 不再是唯一参数来源；关卡特定参数从 `levels.js` 读取，全局常量保留作为默认值
2. **path.js** — `createPath()` 升级为路径类型调度器，原有螺旋线逻辑保留为 `generateSpiralPath()`
3. **main.js** — `ZumaGame` 新增 `currentLevel`、`levelProgress` 字段；`resetRound()` 接受关卡参数；`gameState` 增加 `"levelSelect"` 值
4. **render.js** — 新增关卡选择界面绘制、路径缩略图、通关画面等绘制函数
5. **chain.js / match.js / projectile.js** — 无需修改（规则层不变）

### 不变量

以下不应被 Phase 4 改动：
- `chainHeadS` 位置公式和偏移逻辑
- 三消判定和连锁规则
- splitState 单断链约束
- 消除事件驱动的计分模型
- 碰撞检测阈值

### 渲染缓存注意事项

`staticSceneCache`（背景+轨道+目标的全屏离屏缓存）当前在启动时构建一次。切换关卡后路径改变，必须重建此缓存。`resetRound()` 或关卡加载流程需要调用 `invalidateStaticCache()` 触发重建。同理，`cachedTrackPath`（Path2D 轨道路径）也需要在路径切换后重建。

### 新增模块

- `levels.js` — 关卡配置数组 + 难度参数
- `save.js`（可选）— localStorage 存档读写封装
